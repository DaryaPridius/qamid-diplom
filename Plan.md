# План тестирования приложения "мобильный хоспис"

### Исходные данные:
1. Архив с приложением "Мобильный хоспис";
2. Данные для авторизации (логин: login2 пароль: password2).

### Составляющие приложения
1. Экран загрузки (Splash Screen):
- тематическое изображение;
- анимация загрузки;
- текстовая цитата.
2. Верхняя панель (AppBar):
- иконка "меню", имеющая кнопки перехода к страницам "Главная", "Новости", "Заявки", "О приложении";
- текстовый заголовок с названием приложения "В Хосписе";
- иконка бабочка - переход к странице тематических цитат;
- иконка человека - переход к кнопке "Log out";
- иконка стрелка - возврат к предыдущему экрану на странице "О приложении";
3. Страница авторизации:
- заголовок с текстом "Авторизация";
- поле ввода логина;
 -поле ввода пароля;
- кнопка "Войти".
4. Страница главного меню:
- AppBar;
- блок списка новостей;
- блок списка заявок.
5. Страница новостей "Новости":
- AppBar;
- заголовок страницы с текстом "Новости";
- кнопка сортировки новостей (две разнонаправленные стрелки);
- кнопка фильтра поиска новостей;
- кнопка редактирования новостей;
- список новостей с короткой информацией.
6. Страница заявок "Заявки":
- AppBar;
- загловок страницы с текстом "Заявки";
- кнопка фильтра поиска зявок;
- кнопка добавления новой заявки (Иконка знака "плюс");
- список заявок с короткой информацией.
7. Страница тематических цитат:
 - AppBar;
- заголовок страницы с текстом "Главное - жить любя";
- список цитат.
8. Страница "О приложении":
- AppBar;
- заголовок страницы с текстом "О приложении";
- текстовое описание "Политика конфиденциальности" к ссылке на внешний ресурс;
- текстовое описание "Пользовательское соглашение" к ссылке на внешний ресурс.

### Используемые инструменты:
1. среда разработки Android Studio;
2. Эмулятор Android API 36 (автоматизированное тестирование);
3. Устройство Xiaomi Redmi 9С, Android 10 (ручное тестирование);
4. Espresso Testing Framework - тестовый фреймворк с открытым исходным кодом. Позволяет писать тесовые сценариии пользовательского интерфейса с доступом к исходному коду;
5. Java 11 - Язык написания автотестов;
6. Allure - инструмент для создания отчетов о результатах тестирования в автоматизированных тестовых сценариях.

## Этапы тестирования:

### 1. Определение "границ" приложения и реализованный функционал (4 часа).
- Установить требования для приложения (функциональные и нефункциональные);
- Проверить соответсвие функциональности приложения заданным требованиям.
### 2. Составление чек-листа(6 часов)

- авторизация в приложении;
- проверка главного экрана (отображение новостей, взаимодействие с фильтрами и сортировкой);
- проверка управления новостями (создание,удаление,редактирование новостей);
- проверка отображения тематических цитат;
- проверка раздела "О приложении";
- проверка обработки сбоев сети;
- Проверка при взаимодействии с другими функциями (блокирока экрана, уведомления, звонки).

### 3. Составление тест-кейсов по пунктам чек-листа (11 часов)
### 4. Автоматизация тестирования (14 часов)

В планах автоматизация тестирования следующего функционала приложния:
 1. Авторизация (вход пользователя с валидными и невалидными данными + выход из системы) - тест-кейсы № 1-4;
 2. Новости:
    - создание, редактирование и удаление новостей (тест-кейсы №9,10, 12, 31-33);
    - отображение и сортировка новостей, работа фильтров в разделе (тест-кейсы №5-8)
    - отображение новостей на главном экране (тест-кейс №14).
3. Просмотр тематических цитат (тест-кейсы №17-18).


- Настройка проекта для написания UI-тестов и добавление зависимостей для интеграции с Allure;
- Написание UI-тестов.

### 5. Отчёт о тестировании (2 часа)

- Экспорт отчётов с мобильного устройства(эмулятора);
- Архивирование отчётов.


#### Общее время: 37 часов.
